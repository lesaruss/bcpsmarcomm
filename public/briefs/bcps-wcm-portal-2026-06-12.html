Audience: Sean A. Russell, District Webmaster, Office of Communications. Internal planning record for the WCM Portal build and ADA integration roadmap.

Key Decisions

Dropdown Filter Alignment Fix
The down arrows on the "All Divisions," "All Health," and "All Audits" filter dropdowns sit too close to the right edge. Decision: shift each arrow one space to the left to clean up the appearance before sharing with stakeholders.
Audit Spreadsheet as WCM Data Source
An existing audit spreadsheet will be provided and used to populate WCM names and current audit statuses on each department card. Departments that cannot be mapped due to name changes will be flagged on a reconciliation list rather than left blank or force-mapped.
Department Card Expanded Data Fields
Each department card will display: WCM name (from spreadsheet), audit status (from spreadsheet), ADA score (pending ADA module build), director or executive director name and title, chief name, and a "Visit Website" link. ADA scores remain "Not Assessed" until the module is live.
Click-Through Department Detail Page
Clicking a department card will navigate to an expanded detail page. This page will include a dashboard, a list of submitted support tickets, and ongoing communication threads. Visibility is role-gated: the WCM and their assigned admin see everything; the general public sees a limited view.
Internal Ticketing System Within the Portal
The portal will house its own ticketing system for web-related requests. Ticket history and status will surface on each department's detail page.
Automated ADA Compliance Module
An ADA module will be built into the portal. It will pull each department's known website URL, render it in an iframe alongside an accessibility report, and highlight specific issues inline. Monthly scans will run automatically, with results posted to the WCM's dashboard and delivered via email notification. Because departments are currently on FinalSite (not a K12 Unlocked-managed platform), the report will include specific remediation instructions rather than automated fixes.
Analytics Dashboard Per Department
Each department detail page will include an analytics section scoped to what WCMs actually need to know. Additional analytics tiles will be added over time as the platform matures.
School-Wide Blueprint Potential
The WCM portal is being designed as a blueprint. If BCPS chooses to extend the model to individual schools, the same architecture applies at school level. This is a future option, not a current build requirement.
