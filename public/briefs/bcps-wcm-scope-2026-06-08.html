Key Decisions

WCM Management Controls Moved to a Separate Subpage
The "Add WCM" form and the accept/deny action table will be removed from the main WMC landing page and placed on a dedicated subpage accessible via a clearly labeled button at the top of the module. The main landing page will display: announcements, the audit checklist list, and progress charts only. This separation keeps the primary workflow clean for WCMs who do not need to interact with management controls, and makes training documentation simpler to write.
WCM Management Is Role-Gated, Not Title-Gated
Access to add, approve, or deny Web Content Managers is controlled by the system role "Add Edit Web Content Manager," not by job title. All principals receive this role by default via the principals script. Interim or temporary staff can be granted the role individually as needed. This distinction is critical for help-center documentation.
WCM Training and Certification Status Column to Be Added
A training/certification status column will be added to the WCM table, visible to both admins and WCMs. It will display: certification status (certified or not), completion date, and expiration date. This column serves as the gating indicator for whether a WCM has full access to the live site. The column design will be finalized as needs emerge; the initial three fields are locked as described here.
25-26 Audit Pass/Fail Status via Placeholder Survey Approach
Rather than parsing the full Excel audit data, Yaco D. Abuelafia will create a single one-question placeholder survey for the 25-26 school year. For the 134 schools that passed (confirmed by Vanessa C. Deslandes), a script will populate the yes/pass value. All other schools will receive no/fail. The global pass/fail badge shown in the dashboard will be derived from whether all surveys for the year are at 100%. Tricia A. Allen confirmed this approach as an acceptable compromise.
Checklist Year Context: Test in 25-26, Go Live in 26-27
Current test checklists are built in the 25-26 school year context. The system year auto-switches on July 1, 2026. Yaco D. Abuelafia will manually move test checklist data to the 26-27 year context. No "clone to next year" feature will be built at this time; that functionality will only be built if recurring annual use warrants it.
Checklist Category Labels Using Text Label Question Type
To organize checklist questions by category (e.g., Homepage, School News), the team will add question entries of type "text label" with a sort order of 0 before each grouping. These labels act as section headers within the checklist view without requiring a separate category column in the data model. Vanessa C. Deslandes will apply this pattern when building out the full audit checklist.
Admin Notification on WCM Addition
When a principal or authorized admin adds a new Web Content Manager, the system will send a notification to the admin team. Rodolfo M. Carril surfaced this request as a way to track additions without manually checking school-by-school. Yaco D. Abuelafia confirmed this is buildable and agreed to include it in scope.
Incomplete 2025/26 Audit Items Carried Forward into Checklist
Schools that did not complete all required audit items during the 2025/26 school year will have their outstanding items added directly to the WMC checklist. This allows schools to complete required updates within the existing workflow. Checklist Settings will include a new option to assign audit tasks by individual school name, enabling school-specific task sets rather than a single universal checklist - since outstanding items differ per school.
