This is now the top-level OOC Web Assignment Document. All tasks and queue items are governed from the weekly Friday OOC Web Admin Meeting (Sean + Director) and used to run the Felicia meetings as well. Updated in place after each session, newest entry on top in the Meeting Log tab.

Where we're headed

01 / Self-Service Governance

Move every admin surface off "only Sean can do it" and onto one shared permissions engine, so BCPS staff manage their own content and access without waiting on us.

02 / WCM Certification & Enablement

Grow a certified bench of department and school Web Content Managers through the Pilot Program, recurring Hot Lab working sessions, and a direct feedback loop.

03 / browardschools.ai

Keep the public Powered by AI site current with real district-sourced content, on its own domain, updated in a living tracker before every review meeting.

04 / Web Team Operations Queue

Run one shared queue across the Project Lead track and Felicia's track so nothing falls through, with clear due dates and ongoing responsibilities for each.

05 / Access & Compliance

Extend the shared permissions model to recurring content (meeting series, documents) and start building toward an in-house ADA/accessibility scan capability.

Pillar 01: Self-Service Governance

Documents self-service & request queue Live

All 13 Documents page tiles now run on the shared permissions engine: admins see who has access at a glance and can edit content or grants inline, without a code push. Every viewer can request an edit in plain language; it routes to the document owner for approval, or straight to build if the requester already has edit rights.

Widgets gallery Live

One home for every embeddable module (Find It Fast, Charter School Directory, more to come): live preview, embed code, and per-widget editor access management in one place.

Find It Fast self-service editing Live

Moved off a hardcoded array into a database the assigned editor can update directly. Two live display bugs (pagination, save-button feedback) found and fixed this cycle.

Pillar 02: WCM Certification & Enablement

WCM Pilot Program relaunch In Pilot

Register and certification-course flow rebuilt with its own welcome and account-creation steps. Relaunching Monday, July 27, 2026, with director-facing and participant-facing launch briefs already sent.

Hot Lab for Department WCMs Live

Recurring working session, Tuesdays and Thursdays 11:30am-12:30pm. Twelve real sessions from June-July 2026 are published as gated meeting-note briefs, visible automatically to Web Content Management, District Web Team, and Office of Communications.

In-platform WCM feedback widget Live

WCMs can flag issues or ask questions directly from the pilot pages and course; feedback is stored and queryable today, with routing straight to Sean planned as a fast-follow.

Pillar 05: Access & Compliance

Recurring-document permission model Live

Meeting series (like Hot Lab) now carry shared permissions at the series level, so every past and future session inherits access automatically instead of needing to be re-granted each time.

In-house ADA / accessibility scanning Queued, due 7/31

Prototype already exists, it's the same tool used for the department scans. Next layer planned: surface real plain-language guidance behind "more info" instead of just linking out. Confirmed as a faster in-house alternative after AudioEye couldn't rescan for 45 days and Finalsite's own renewal looks unlikely.

Pillar 06: Production Systems (New)

Graphics-request AI production agent Workshopping

Departments submit a graphics request through a wizard tied to their Marcom account, an AI agent drafts it, a human graphics staffer reviews/edits inside a dashboard, and delivery happens back through the same console with a live notification instead of email. Still an idea being workshopped with the Director before wider rollout; a VistaPrint print-fulfillment integration was floated as a later addition.

Social media dashboard concept Queued
