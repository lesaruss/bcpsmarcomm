Key Decisions

v1 Brief Superseded - Context Must Drive the Document
Yaco D. Abuelafia reviewed the June 8 brief and identified that it lacked the technical specificity discussed in prior meetings. The team agreed the brief should reflect the actual development context, not a generic summary. This v2 document is the authoritative scope reference going forward.
Certification Module: 4 Statuses Defined, V1 Must-Ship by June 30
Vanessa C. Deslandes-Auguste's roadmap designates WCM Certification Status Tracking as a V1 must-ship by June 30, 2026. Yaco D. Abuelafia to add the certification status column to the WCM table with four defined statuses: Not Registered (default), Registered (enrolled), Certified (completed), and Incomplete (started but not finished). The module should be designed for district-wide reuse. WCM certification training deadline is September 30, 2026.
3-Status Audit System Confirmed: Pass / Incomplete / Fail
The run/pass/fail script requires three distinct statuses, not two. Pass: all surveys for the school are 100% complete and district-approved. Incomplete: one or more surveys remain unfinished. Fail: the school has not started any surveys. Vanessa C. Deslandes-Auguste confirmed this distinction. The data migration strategy for pre-existing 2025-26 records is still under analysis by Yaco D. Abuelafia.
School-Specific Incomplete Checklists - Each School Has Different Outstanding Items
Vanessa C. Deslandes-Auguste confirmed that incomplete schools from the 2025-26 audit do not share the same outstanding items - each school's remaining tasks are different. The agreed approach: create a master checklist template with all audit items, then allow Vanessa to mark completed items per school so only the true outstanding tasks are visible to each school. Yaco D. Abuelafia proposed an alternative via programmatic bulk-marking of completed items by school location number.
June 30 Deadline Requires Priority Ranking
Yaco D. Abuelafia stated clearly that not all scope items can be completed by June 30, 2026. The team began prioritizing at the end of the session. Tricia A. Allen identified the 2025-26 audit pass/fail record as the top priority. Email automation (except the welcome email) was deferred to a post-June 30 phase. Yaco will complete the pass/fail script and priority items first.
Three-Phase Roadmap Established: V1 (June 30), V2 (August 30), V3 (October 2026)
Vanessa C. Deslandes-Auguste provided a formal phased roadmap after the June 9 session. V1 (June 30): WCM assignment notification, 2025-26 audit status display, WCM certification status column. V2 (August 30): Audit dashboard enhancements (by region/school, filter/sort, PBI export), 2026-27 audit and update notifications, 2025-26 incomplete audit follow-up with automated reminders. V3 (October 2026): Automated certification recognition, digital badge, certificate, and Recognition and Achievement Dashboard.
Group-Based Checklist Assignment Needs Definition Before Build
Yaco D. Abuelafia raised the possibility of assigning checklists by school type or group (e.g., high schools get 10 items, elementary schools get 6). Vanessa C. Deslandes-Auguste confirmed this is a likely future need. This feature is not in scope for June 30 - it requires Vanessa to define which school groups receive which checklist sets before development can begin.
